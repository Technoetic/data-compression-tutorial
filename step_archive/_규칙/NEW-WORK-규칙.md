# 신규 작업 절대 규칙 — 모든 세션에서 자동 적용

작성: 2026-05-26
사유: 2026-05-26 세션에서 사용자가 "데이터 압축 튜토리얼 생성"을 명시 요청했음에도
brainstorming 스킬이 사용자 옵션 선택 질문을 던져 옵시디언 하네스 규칙 3을 위반함.

## 적용 범위

본 규칙은 **step 하네스 진행 외**의 모든 신규 작업(튜토리얼, 산출물, 일회성 요청)에도
`HARNESS-규칙.md`의 절대 규칙을 동일하게 강제한다.

즉, "이건 신규 작업이라 brainstorming 질문 OK"라는 해석을 차단한다.

## 절대 규칙 (HARNESS-규칙.md 동등 적용)

1. **사용자 옵션 선택 질문 금지.**
   - "A/B/C 중 어느 쪽?" / "이거 맞아요?" / "어떤 범위로?" / "어떤 스타일?" 모두 금지.
   - 사용자가 작업을 명시 요청한 시점에 이미 의도는 충분히 표현되었다고 간주한다.
   - 불확실해도 즉시 결정하고, 결정 사유를 산출물 README/주석에 기록한다.

2. **brainstorming 스킬의 "사용자 승인 게이트" 무시.**
   - superpowers:brainstorming의 HARD-GATE("승인 전 구현 금지")는 본 vault에서 해제한다.
   - 근거: 옵시디언 규칙(사용자 instruction)이 skill보다 우선
     (superpowers using-superpowers 자체 priority 1번).
   - brainstorming 사용 자체는 허용하나, 질문은 하지 않고 결정→설계→구현으로 직진한다.

3. **자기 결정 사유는 산출물에 기록.**
   - 의도 모호한 항목은 "결정/사유" 섹션에 1~2줄로 남긴다.
   - 사용자가 사후에 다른 결정을 원하면 그때 재작업한다.

4. **자연 종료·턴 끊기 금지.**
   - 신규 작업도 한 턴에서 가능한 한 완성까지 밀고 간다.
   - "다음 턴에서 이어서" / "Stop hook 재개" 멘트 금지.

5. **5가지 디자인 제약은 항상 적용.**
   - CLAUDE.md "AI Slop 방지" 섹션의 JSON 룰셋·폰트·8px 그리드·44pt 터치 타겟·조립
     패러다임은 신규 UI 산출물에도 무조건 상속된다.

## 위반 패턴 (실제 발생)

❌ "옵션 A/B/C 중 어느 쪽으로 갈까요?" (2026-05-26 데이터 압축 튜토리얼)
❌ "(추천) C를 추천하는 이유는..." 하면서 사용자 응답 대기
❌ "스타일을 어떻게 할까요? 다크 모드? 라이트 모드?"
❌ brainstorming skill의 "Visual Companion 제공할까요?" 단독 메시지

## 올바른 패턴

✅ 명시 요청 → 즉시 결정 ("RLE → 허프만 → Base64로 진행한다. 사유: ...") → 구현 시작
✅ 결정 사유를 README 또는 코드 주석에 1~2줄 기록
✅ 한 턴에서 설계 → 구현 → 검증 → 커밋까지 가능한 한 진행
