# Step 030 — 레이아웃 설계 대안 3종 (ToT)

## Step-Back

- **해결할 핵심 문제**: 4개 알고리즘 단원이 일관된 학습 패턴(입력 → 시각화 → 결과 → 컨트롤)을 가지면서, 단원마다 다른 시각화 메타포(픽셀 격자 vs 트리 vs 테이프 vs 비트 매핑)를 자연스럽게 담는 레이아웃 결정.
- **제약**: 단일 페이지 + 다크 OLED + 단일 액센트 + 8px 그리드 + 44pt 터치 + 외부 의존성 0 + 한국어 본문.

## 대안 A. "수직 스토리보드" (linear.app + ciechanow.ski 혼합)

```
┌─────────────────────────────────────────────────────────┐
│ Header (sticky, 56px) │ 앵커 네비 (5개)                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  §1 RLE                                                 │
│  ──────────                                             │
│  [개념 텍스트 (좌 30%)] │ [위젯 (우 70% 풀폭)]            │
│                                                         │
│  → 100vh 자연 분량, 자연 스크롤                          │
├─────────────────────────────────────────────────────────┤
│  §2 Huffman                                             │
│  ... (반복)                                             │
└─────────────────────────────────────────────────────────┘
```

**장점**: linear-style 거대 섹션. 시각적 인상 강함. 모바일 reflow 쉬움.
**단점**: 단원 간 비교가 어려움(스크롤 거리 큼). 학습 흐름이 단조로움.

## 대안 B. "단원 카드 + 사이드 네비" (visualgo + distill 혼합)

```
┌──────────────────────────────────────────────────────┐
│ Header                                               │
├────────┬─────────────────────────────────────────────┤
│ Aside  │ Main Content                                │
│ (240px)│                                             │
│ §0 ●   │ ┌──────────────────────────────────────┐    │
│ §1     │ │ §1 RLE Card                          │    │
│ §2     │ │ [개념] [입력] [위젯] [결과] [컨트롤] │    │
│ §3     │ └──────────────────────────────────────┘    │
│ §4     │ ┌──────────────────────────────────────┐    │
│ §5     │ │ §2 Huffman Card                      │    │
│        │ └──────────────────────────────────────┘    │
└────────┴─────────────────────────────────────────────┘
```

**장점**: 단원 점프 빠름. 데스크톱에 적합. 학습 진행 상태(●) 표시 가능.
**단점**: 모바일에서 사이드 네비 충돌. 외부 의존성 0 정책상 햄버거 자체 구현 필요.

## 대안 C. "수직 스토리보드 + 상단 sticky 진행률" (자체 추천)

```
┌─────────────────────────────────────────────────────────┐
│ Header (sticky, 56px)                                   │
│ ─ Progress: [●●●○○○] §3 LZ77 ──────────────────────── │
├─────────────────────────────────────────────────────────┤
│ §0 인트로 ─ 정보 엔트로피                                │
│    "압축은 왜 가능한가?"                                 │
│    [엔트로피 미터: 사용자 입력 → 비트/문자]              │
├─────────────────────────────────────────────────────────┤
│ §1 RLE                                                  │
│  ┌─[헤더] 단원명 + 한 줄 정의 + 대중 앱 사례 칩───────┐  │
│  │ [개념 (텍스트)]   │ [입력 textarea]               │  │
│  │ [시각화 (그리드)] │ [결과 (압축률 막대 + 토큰)]   │  │
│  │ [컨트롤: Reset / Step ▶ / Auto ▶▶ / 인스펙터]    │  │
│  └─────────────────────────────────────────────────────┘
│                                                         │
├─────────────────────────────────────────────────────────┤
│ §2 Huffman ...                                          │
└─────────────────────────────────────────────────────────┘
```

**장점**:
- 단일 페이지 스크롤(linear-style)의 임팩트.
- 진행률 바로 학습 진척 표시(distill·visualgo 차용).
- 단원 카드 내 4분할은 모바일에서 1열 reflow.
- 외부 의존성 0.

**단점**: 단원 간 점프는 진행률 막대를 클릭해야 함(=앵커 네비). 약간의 햄버거 대체 필요.
